### C-1 — The arming contract (D-34, the invisible-page landmine) — READ FIRST

Phase 1 shipped `[data-reveal]` attributes **inert** with **zero `[data-reveal]` rules in `styles.css`**, on purpose (`01-03-SUMMARY.md:221`, warning at `styles.css:550-555`). `updateReveals()` is what sets `opacity:0`. **Any CSS that hides a reveal element before or without JS ships a fully invisible page to every visitor.**

**THE RULE, in one line:**

> **No hiding rule may key on a selector that exists without JS. `hidden` and `shown` must be two values of one JS-created attribute, at identical specificity.**

**Contract — `data-rev-state`:**

- JS writes **only** `el.dataset.revState = 'hidden' | 'shown'`. CSS keys **only** on `[data-rev-state="…"]`.
- JS never runs → the attribute never exists → **no rule matches** → the page renders fully. D-34 satisfied *structurally*, not by discipline.
- `hidden` and `shown` are both `[data-rev-state="…"]` → specificity (0,1,0) vs (0,1,0). **The cascade cannot be gotten backwards.**
- The **same synchronous pass** arms and observes (`initReveals()`). The code that hides is the code that shows — that is V4's real insight, not the fade.

**⚠ This SUPERSEDES the sketch's `body.js` pattern — deliberately.** Sketch 001 scoped hiding under `body.js` and it **shipped a dark page twice** (`001-method-section/README.md` § "Bugs found", bug 1): scoping the hide under `body.js` made it `(0,3,1)`, out-specifying the `.in` reveal at `(0,3,1)`, so the class was added and the CSS ignored it. The fix was to carry `body.js` on *both* sides to reach `(0,4,1)` vs `(0,3,1)`. **That fix works but is a specificity race a future edit can silently lose.** `data-rev-state` removes the race entirely. Use it.

**Applies to METHOD's bespoke choreography too.** METHOD's internal elements (`.ph__wire`, `.ph__desc`, `.doc-rail`, `.key`) are **not** `[data-reveal]` elements — they are driven by a timer sequence, not the observer. They get the same rule shape, not `body.js`:

```css
/* JS sets data-method-state on the section. Same contract, same specificity symmetry. */
[data-method-state="armed"] .ph__wire { transform: scaleX(0); }
[data-method-state="playing"] .ph__wire.in { transform: scaleX(1); }
```

**Verification gate (must be in the plan):**
1. `grep` the `styles.css` diff for the bracketed reveal selector **without** `rev-state` → any hit fails the gate.
2. Disable JS in the browser → **every word of the page must still be readable.** This is a hard gate, not a nice-to-have.
3. `grep` the diff for `body.js` → should be **zero hits** in `site/`.

### C-2 — The reveal vocabulary (`data-reveal-kind`)

One attribute, four behaviours. **JS owns state; CSS owns every duration, easing and transform.** This is what makes MOTION-02 free (D-36) — the reduced-motion block at `styles.css:1156` can reach CSS transitions; it cannot reach a JS string. V4 writes `el.style.transition = 'opacity 1s ease …'` (`V4:566`) — **do not port that plumbing.**

| `data-reveal-kind` | Applies to | Mechanism | Cost |
|---|---|---|---|
| `draw` | `.section__rule` (`styles.css:645`), `.exp__spine` (`:953`), METHOD's `.ph__wire` + loop arc | `transform: scaleX(0)→1` (`scaleY` for the spine) + `transform-origin` | **FREE** — compositor-only |
| `assemble` | ORIGIN's 3 beats, METHOD's 5 manifesto stanzas, `h2` | `overflow:hidden` frame + inner child `translateY(100%)→0` | **CHEAP** — transform + one clip |
| `snap` | `.exp__dot` (`:979`), `.pill`, `.plate__meta`, METHOD's `.ph__dot` | `transform: scale(.6)→1` + opacity, overshoot | **FREE** |
| *(absent)* | everything else | V4 fade + `translateY(30px)`, **blur removed** | **CHEAP** |

**Two deliberate deviations from D-35's "port it as the baseline" — named, not silent.** Both sit inside CONTEXT.md's Claude's-Discretion grant (*"how the reveal is wired — thresholds, stagger, re-arm"*). V4's **behavioural** contract (stagger, re-arm on exit, reduced-motion branch) is preserved 100%; only its plumbing changes:

| # | Deviation | Why | Evidence |
|---|---|---|---|
| DEV-1 | **Trigger:** `scroll` + per-element `getBoundingClientRect()` → **IntersectionObserver** | V4 writes styles then reads rects **per element, per scroll event** — textbook layout thrashing, ~20 forced reflows/scroll on a page also running a canvas rAF loop. IO resolves geometry in the browser's own layout phase. **This honours D-37 more strictly than the original does.** | `V4:569`, `:587`; research Pitfall 2 |
| DEV-2 | **Drop `filter: blur(10px)→blur(0)`** | Chrome's own guidance: *"don't animate the blur value directly."* Blur is a convolution; V4 staggers several overlapping, on a page already spending GPU on two canvases + a `backdrop-filter` glass disc. `assemble` and `draw` read as **more** deliberate at ~zero cost. Also lets us drop V4's `setTimeout(…, 1700+d)` cleanup (`V4:594`). | `V4:575/583/593`; research Pitfall 3 |

**⚠ Never call `getBoundingClientRect()` inside the IO callback** — that reintroduces the exact reflow DEV-1 removes. Use `entry.isIntersecting`, which is pre-resolved and free.

### C-3 — Timing: **SLOW**. This is the phase's hardest-won lesson.

> Sirio **could not read** the sketch's first cut. Timings were roughly **tripled** (~7s → ~21s). The MANIFEST records it as a phase-wide lesson: **"reveal timings tuned by eye run far too fast."** Treat every number below as a floor, and expect Sirio to want them slower still, not faster.

**Stagger is a token, not a JS constant.** JS writes the index; CSS owns the interval. This makes re-pacing the whole page a one-line change — which, given the lesson above, is the point.

```css
:root { --rev-stagger: 130ms; }                    /* V4's approved interval — furniture default */
[data-rev-state] {
  transition-delay: calc(var(--rev-stagger) * var(--rev-i, 0));
}
#origin, #method { --rev-stagger: 220ms; }         /* narrative beats: slower, they are read, not scanned */
```

`initReveals()` sets `el.style.setProperty('--rev-i', el.dataset.reveal || 0)` — unitless. **JS never computes milliseconds.**

|---|---|---|---|
| default fade-up | `1s` | `cubic-bezier(0.22, 1, 0.36, 1)` | V4:566, approved |
| `assemble` | `1s` | `cubic-bezier(0.22, 1, 0.36, 1)` | matches default — one curve for the page |
| `draw` | `900ms` | `cubic-bezier(0.22, 1, 0.36, 1)` | sketch 001 variant D |
| `snap` | `450ms` | `cubic-bezier(0.34, 1.56, 0.64, 1)` (overshoot) | research; the dot *pops* after the spine passes |
| Stagger — furniture | `130ms × index` | — | V4, approved |
| Stagger — narrative | `220ms × index` | — | **this spec** — the "slow" lesson applied |

## NAV-01 — The Full-Height Contract

**Requirement:** *"Immersive single-scroll home with distinct, full-height sections"* (`REQUIREMENTS.md:23`).
**Interpretation (Claude's discretion, per CONTEXT.md):** **a floor, not a cage.** "Distinct", not "paginated".

**Phase 1 already solved this on the hero. Reuse it verbatim — do not invent a second approach.**

```css
/* NAV-01 — mirrors the shipped cascade at styles.css:164-165 exactly.
   min-height is a FLOOR: content that overruns simply grows. That is the
   property that makes this survive the Phase 3 mobile pass unchanged. */
.section--full {
  min-height: max(100vh,  620px);   /* fallback for engines without svh */
  min-height: max(100svh, 620px);   /* overrides where svh is supported */
  display: flex;
  align-items: center;
}
.section--full > .section__inner { width: 100%; }
```

| Section | Floor | Why |
|---|---|---|
| `#top` hero | ✅ already | shipped, `styles.css:164-165` |
| **`#origin`** (new) | ✅ **add** | narrative interlude — **the pause is the point** (D-02: unnumbered, quieter) |
| **`#method`** (new) | ✅ **add** | narrative interlude; the section Sirio asked to be surprised by |
| `#index`, `#publications`, `#contact` | ⏸ **named open item** | research recommends floors on all three; **this spec narrows to the interludes this phase builds.** See § Open Items OI-3 — a one-line class each, decided on a render, not silently dropped |
| `#work`, `#experience` | ❌ never | 3 plates / 4 rows ≫ one viewport. A floor is a no-op anyway |

**`svh`, NOT `dvh` — a specific trap for *this* page.**

- `100vh` resolves against the **large** viewport (chrome hidden). On mobile load the chrome is *visible* → a `100vh` element **bleeds off-screen**.
- **`svh`** — smallest viewport, chrome visible. Fixed at load; **never reflows.** ✅
- **`dvh`** — ⛔ resizes as the URL bar shows/hides, i.e. **a layout reflow fires during the exact scroll gesture that triggers reveals**, on top of a live canvas rAF loop. Most sites can afford `dvh`; a page with a live canvas loop cannot.
- `svh`/`lvh`/`dvh` are **Baseline Widely Available since Jun 2025**.
- ⛔ **Do NOT use the `--vh: innerHeight * 0.01` JS hack** — it needs a resize listener, i.e. more scroll-time work on the exact device it claims to help.

**⛔ Explicitly NOT scroll-snap.** `y mandatory` on a page with variable-height sections (Work is ~3 viewports) traps the reader mid-section; it fights iOS momentum; it is user-agency-hostile in a way that reads as **broken, not polished** — the opposite of the Core Value. Distinctness is already delivered by the `--accent` system + the floor + `scroll-margin-top: 70px` (`styles.css:565`).

## Typography

### T-1 — The reality, and the deviation from the template

⚠ **The template asks for 3–4 fixed sizes and exactly 2 weights. The live site is fluid `clamp()` across ~12 roles at weights 300/400/500.** This spec **documents what exists and defends it**; it does **not** retro-fit a fake fixed scale onto a shipped fluid system, and it does **not** rewrite the live site's tokens to satisfy a template.

**Why fluid is right here and the deviation stands:**
1. **The page is one immersive scroll on a cosmic canvas.** Type that steps at breakpoints while a starfield scales continuously reads as broken. Fluid type is the only kind that matches the medium.
2. **`clamp()` has a floor and a ceiling** — `clamp(40px, 6vw, 86px)` is *more* constrained than "one of four sizes", not less. It is a bounded range, not freedom.
3. **Phase 3 is the mobile pass.** Fluid type is what makes that pass a tuning job rather than a rebuild. Fixed sizes would guarantee a breakpoint matrix.
4. **It is already shipped and approved.** Changing it is out of scope and would touch every section.

**Weights: {300, 400} for all new work.** The live site also uses 500, on exactly one element — `.section__numeral` (`styles.css:596`) — which the interludes **do not have** (D-02). So the new sections land on **exactly two weights**, satisfying the template's intent through the design, not around it.
⚠ **The sketch used `700` for `.track__kind` (DIGITAL/PHYSICAL). Pull it back to 400.** The live site never uses 700. Let the **accent colour and the 0.24em tracking** carry the emphasis — that is how every other label on the page does it.

## Spacing

### S-1 — The reality, and the deviation from the template

⚠ **The template requires "multiples of 4 only". The live site's section rhythm is fluid: `--pad-section: clamp(80px, 12vh, 150px) clamp(20px, 6vw, 84px)` (`styles.css:37`).** `12vh` does not resolve to a multiple of 4 and cannot — that is the point of it.

**This spec declares a two-tier contract that is honest about both:**

| Tier | Scale | Applies to | Multiples of 4? |
|---|---|---|---|
| **Macro — fluid** | `clamp()`, viewport-relative | section padding, section-head margins, inter-block rhythm | **No, deliberately.** The rhythm must breathe with the viewport, for the same reason the type does (§ T-1). `clamp()` bounds it at both ends |
| **Micro — fixed** | **4, 8, 12, 16, 20, 24, 32, 48** | gaps, element padding, dot/wire offsets, label gaps — **anything the viewport should not move** | **Yes, enforced for all new work** |

**Why the macro tier is right, not sloppy:** a fixed 64px section gap that looks generous at 1440px is claustrophobic at 2560px and enormous at 375px. `clamp(80px, 12vh, 150px)` is a **bounded** range with a hard floor and ceiling — it is a *constraint*, not an absence of one. The 4px grid exists to stop arbitrary values; `clamp()` with explicit bounds achieves the same discipline against a moving reference frame.

**Where the micro tier binds — enforced this phase:** the sketch used `26px`, `22px`, `10px`, `6px` freely. **Normalize all new fixed spacing to the 4-multiple scale.** No exceptions requested; none granted.

### S-2 — Inherited rhythm (reuse, do not re-derive)

| Token / value | Anchor |
|---|---|
| `--pad-section: clamp(80px,12vh,150px) clamp(20px,6vw,84px)` | `styles.css:37` |
| `scroll-margin-top: var(--nav-h)` = 70px — anchors land clear of the docked nav | `styles.css:565` |
| `--max-w: 1180px` on `.section__inner` | `styles.css:38`, `:628` |
| `.section__head` gap: `20px` | `styles.css:635` |
| `h2` margin-top: `14px` | `styles.css:656` |

⚠ `14px` (the shipped `h2` margin) is **not** a multiple of 4. It is **inherited and out of scope** — do not "fix" it. The micro-tier rule binds **new** work only.
