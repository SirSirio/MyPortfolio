<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer laptop -> GitHub | git push + repo/Pages configuration crosses into a third-party host |
| GitHub Actions runner -> Pages CDN | build artifact (static files) published to public web |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | CI supply chain (`.github/workflows/deploy.yml` actions) | mitigate | Use only pinned first-party `actions/*` (checkout, configure-pages, upload-pages-artifact, deploy-pages); no third-party marketplace actions |
| T-01-02 | Elevation of Privilege | Workflow token scope | mitigate | Scope `permissions` to the minimum (`contents: read`, `pages: write`, `id-token: write`); no write to repo contents |
| T-01-03 | Information Disclosure | Repo contents published to public Pages | accept | Only `app/` is uploaded as the artifact; planning docs and profile.md stay out of the artifact path. No secrets exist in this static project |
| T-01-SC | Tampering | package installs (npm/pip/cargo) | accept | This plan installs NO packages (plain static files, no bundler, no package.json) — package legitimacy gate not applicable |
</threat_model>
