# Quick Task 260717-oj6 — EndoSense gif into its media slot, clickable to the iGEM wiki

## Goal

Fill the EndoSense plate's media slot (currently the striped `[ project image ]`
placeholder at `site/index.html:251`) with `Media/EndoSense.gif`, and make the
image a link to `https://2024.igem.wiki/DTU-Denmark/` that opens in a new tab.

## Context / constraints

- Media convention (`site/assets/media/README.md`): assets are **committed to the
  repo** and referenced by **relative path** — no hot-linking. Naming is lowercase,
  hyphenated, matching the project slug → `endosense.gif`. Source lives at
  `Media/EndoSense.gif` (1920×1080 = exact 16:9, matches the slot's aspect ratio).
- The slot is normally a **drop-in plus one line, no CSS change**. The placeholder
  auto-hides via `.media-slot:not(:has(> img, > video))` and a filled asset is
  styled by `.media-slot > img, .media-slot > video` — both **direct-child**
  selectors. Wrapping the image in an `<a>` to make it clickable makes the `<img>`
  a *grandchild*, so those selectors stop matching: the placeholder would not
  switch off and the image would not fill the box. So "clickable" requires a
  small, scoped CSS generalization. This is the same kind of acknowledged
  deviation as `.media-slot--composite`, but narrower (adds an `a > img` case,
  changes no geometry, weakens neither the first-party nor no-stock/no-AI rules).

## Tasks

1. **Add the asset.** Copy `Media/EndoSense.gif` → `site/assets/media/endosense.gif`
   (committed, relative-path-referenceable).
   - verify: file exists at `site/assets/media/endosense.gif`, non-zero, GIF 1920×1080.
   - done: asset in repo under the media folder with the slug-matching name.

2. **Fill the slot, clickable.** In `site/index.html:251`, put an anchor-wrapped
   `<img>` inside the EndoSense `<figure>`:
   `<a href="https://2024.igem.wiki/DTU-Denmark/" target="_blank" rel="noopener"><img src="./assets/media/endosense.gif" alt="…"></a>`.
   Keep `data-label` in place (per README — it returns automatically if the asset
   is removed).
   - verify: figure contains an `<a>` → `<img>`; href/target/rel correct; src is the
     relative path.
   - done: slot markup filled with the clickable image.

3. **Generalize the media-slot CSS** (`site/styles.css`, ~1647 / ~1667) so an
   anchor-wrapped image counts as "filled":
   - Empty-state selectors (`.media-slot:not(:has(> img, > video))` and its
     `::after`) → also treat `> a > img` as filled: `:not(:has(> img, > video, > a > img))`.
   - Fill rule → add `.media-slot > a > img` alongside `> img, > video`, and make
     `.media-slot > a` a block filling the slot so the whole image is the hit-target.
   - verify: with the gif in place the stripes/label/dashed border are gone, the gif
     covers the 16:9 box, and clicking anywhere on it opens the wiki in a new tab.
   - done: clickable image renders correctly; every other (empty/composite) slot
     unchanged.

## Must-haves

- `site/assets/media/endosense.gif` committed (relative path, first-party, no hot-link).
- EndoSense slot renders the gif, placeholder chrome gone.
- Clicking the image opens `https://2024.igem.wiki/DTU-Denmark/` in a new tab
  (`target="_blank"`, `rel="noopener"`).
- No regression to empty slots or the composite slot.

## Out of scope
